About This Bot

Why I built this

I built this. It's a paper-trading bot for crypto — BTC, ETH, and SOL — that trades against real, live market prices with simulated money, using rule-based strategies that have to prove themselves on real historical data before they're allowed anywhere near a live decision. No strategy goes live on a hunch. No fill, price, or profit in this system is ever fabricated — if the bot can't get a real number, it says so and skips the cycle instead of making one up.

I built it with Claude (Anthropic) as my AI engineering partner — not as a black-box "AI trading brain" bolted onto the front, but as the tool I used to design, implement, test, and review every module in this codebase, task by task, with real test evidence at every step. This document is that system, written down completely enough that anyone on the AFTERLEXX team could stand up an identical instance from scratch.

01Overview

Runs a continuous paper-trading loop against three cryptocurrencies, using real live prices, real (simulated) fees and slippage, and rule-based strategies that are only allowed to trade if they pass a real backtest first. Every few hours it re-checks its own strategies against fresh data and promotes or demotes them automatically — that's the "learns from its mistakes" behavior. It does not execute real trades, hold real funds, or connect to any exchange account.

Assets tradedBTC/USDT, ETH/USDT, SOL/USDT (Binance spot pairs)
ModePaper trading — simulated cash, real prices
Starting balance$10,000.00 (simulated)
Data sourceBinance public REST API — no API key required
Language / runtimePython 3.9
PersistenceSQLite — single local .db file
DashboardStreamlit + Altair
StrategiesSMA crossover, RSI threshold, Momentum — rule-based, no runtime LLM inference
Position sizingHalf-Kelly, from each strategy's real backtested win rate & payoff ratio
Poll intervalEvery 300 seconds (5 minutes)
Re-vet intervalEvery 72 cycles (~6 hours)
AI's roleDevelopment partner — design, implementation, testing, review. Not a runtime trading component.

03Core Build Principles

The actual operating rules I held Claude to throughout development — the real instructions that shaped every module in Section 04, not illustrative examples. Every commit in this codebase was built and reviewed against these.

RULE — Never fake a fill, a price, or a profit Applies to every module that touches money or market data. If a real price/fill/fee can't be obtained, the system says so and skips that unit of work — never substitutes a guessed or last-known-good value and presents it as current.
RULE — Verify, don't guess Every constant, formula, and API assumption was checked against real API responses or official docs before being written into code. Where a value genuinely couldn't be verified, it's explicitly disclosed as an assumption, not presented as fact.
RULE — A strategy only trades live if it earns it No strategy is wired into the live loop directly. It must pass a real backtest against real historical candles first, and it's re-checked periodically — passing once is not permanent.
RULE — Every task ships with real test evidence Each module was built test-first, with actual pytest output attached to every commit, plus an independent code review pass before being marked done.

Three independent, deterministic, rule-based strategies. Each implements signal(closes) -> "buy" | "sell" | "hold".

SMA Crossover — fast=10, slow=30 
if fast_prev <= slow_prev and fast_now > slow_now: return "buy"
if fast_prev >= slow_prev and fast_now < slow_now: return "sell"

if rsi_prev >= overbought and rsi_now < overbought: return "sell"
Momentum — lookback=20, threshold=0.0 
ret = (closes[-1] - closes[-(lookback+1)]) / closes[-(lookback+1)]
if ret > threshold: return "buy"
if ret < -threshold: return "sell"

4.3 — Risk Layer tradingbot/backtest/ + tradingbot/sizing/

Backtest gate — a strategy earns the right to trade live:

FEE_RATE = 0.001  // 0.1%, same as live taker fee

passed = sharpe > 0 and max_drawdown < 0.5 and num_trades >= 2

Position sizing — half-Kelly, from the strategy's own real backtested trade outcomes:

Half-Kelly Position Sizing 
def kelly_fraction(win_rate, payoff_ratio):
    p, q, b = win_rate, 1 - win_rate, payoff_ratio
    if b <= 0: return 0.0
    return max(0.0, min((b*p - q) / b, 1.0))

def position_size(equity, win_rate, payoff_ratio, fraction=0.5):
    return equity * kelly_fraction(win_rate, payoff_ratio) * fraction

4.5 — Re-vetting Layer tradingbot/loop/main.py

The "learns from its mistakes" behavior. Every 72 cycles (~6 hours), every strategy is re-backtested against fresh 500-candle hourly history:

Periodic Re-vetting 
REVET_INTERVAL_CYCLES = 72  // 6 hours at the 300s poll interval

def maybe_revet(conn, cycles_since_vet, approved_strategies, ...):
    cycles_since_vet += 1
    if cycles_since_vet < revet_interval:
        return approved_strategies, cycles_since_vet
    try:
        new_approved, _ = vet_fn(conn)
    except (PriceFetchError, RequestException):
        // keep current approvals, retry next interval — never crash
        return approved_strategies, 0
    return new_approved, 0

A strategy that starts failing gets demoted — blocked from new positions, but still allowed to close any position it already holds. A previously-rejected strategy that starts passing gets promoted.

05Step-by-Step Operational Guide

The full cycle, exactly as tradingbot/loop/main.py executes it every 300 seconds.

  1. Startup vetting — on process start, fetch 500 hourly candles per symbol, backtest all 3 strategies against each, log every result to strategy_runs, build the initial approved list.
  2. Seed price history — reuse the same 500-candle series as the starting live signal history.
  3. Poll live prices — fetch current price per symbol. On failure, skip that symbol this cycle only — never substitute a stale price.
  4. Append to price history — each successful price grows that symbol's running series.
  5. Build the evaluate-set — currently-approved combos, plus any demoted combo that still has an open position.
  6. Evaluate signals — call strategy.signal(closes) for each combo in the evaluate-set.
  7. Act on buy signals — if approved and no position is open: size via half-Kelly, open the trade.
  8. Act on sell signals — if a position is open: close it, regardless of current approval status.
  9. Log equity — compute total portfolio equity, insert one row into equity_history.
  10. Re-vet check — every 72 cycles, re-run vetting and replace the approved list (errors keep current approvals, never crash).
  11. Sleep — wait 300 seconds, repeat from step 3.

Any unexpected exception in a single cycle is caught at the top level and logged — it never kills the loop process.

07Rebuilding From Scratch

  1. Set up the four core packages: data/ (market data), strategies/ (signal generation), backtest/ (pass/fail gate), sizing/ (Kelly position sizing).
  2. Build engine/ (portfolio state, broker fill logic, SQLite schema/CRUD) on top — the only layer that mutates money state.
  3. Wire it together in loop/main.py following the 11-step cycle in Section 05 exactly — vet once at startup, then poll / evaluate / act / log / sleep, with periodic re-vetting layered on top.
  4. Build dashboard/app.py as a read-only consumer of the same SQLite file, plus its own single live price call for unrealized P&L display.
  5. At every step, hold to Section 03's four rules — especially "never fake a fill, a price, or a profit." That rule is what separates this from a toy backtest dressed up as a live system.
